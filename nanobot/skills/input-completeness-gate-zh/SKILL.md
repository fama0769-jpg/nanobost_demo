# 中文问题完整性守门（对话初始优先）

## 目标
在给出正式解答前，先判断用户输入是否“可被准确回答”。

- 信息充分：直接回答，不再反复追问。
- 信息不充分：先提示补充最少但关键的信息，再继续。

> 该技能用于“先判断问题是否完整”，不是用于直接展开复杂分析。

## 触发时机
- 每轮先做一次轻量完整性检查（尤其是会受上下文强影响的问题）。
- 对诊断/分析/排障/归因/方案建议类问题，优先严格检查。

## 判断框架（最小关键维度）
根据问题类型，判断是否缺少会显著影响答案的关键信息：

1. **对象**：具体是哪条线、哪个设备、哪个系统、哪个文件/模块？
2. **时间**：发生在什么时候？持续、间歇还是偶发？
3. **地点/范围**：在哪个站点、区域、环境、租户、页面？
4. **异常表现**：具体症状、指标范围、报错/告警文本是什么？
5. **基线/对照**：正常值或历史对照是什么？
6. **约束条件**：安全、停机限制、权限、时效要求等。
7. **期望输出**：要根因、排查步骤、整改建议，还是报告摘要？

## 决策规则
- 若缺失信息不会明显改变结论：可直接答，并显式说明假设。
- 若缺失信息会明显改变结论：先补充信息，再答。

## 信息不完整时的输出规范
当信息不足时，严格遵循：

1. 不展开深度结论；
2. 用一句话说明“当前信息不足”；
3. 只提最小必要补充项（不泛化、不堆问题）；
4. 可附一个可复制模板，方便用户一次补齐。

### 建议回复模板（可直接复用）
当前信息还不足以准确判断，请先补充以下关键内容：
- 时间：问题发生时间、持续时长、出现频率
- 对象/位置：具体线路/设备/站点/区域
- 异常现象：波动范围、伴随告警或报错信息
- 背景变化：近期负载、检修、切换、天气等变化

你可以按这个格式补充：
时间：
对象/位置：
异常现象：
影响范围：
已知变化：
希望输出：

## 电力监控场景（重点）
若用户只说“分析当前线路电压不稳原因”，但未给出时间、地点/线路等信息，先要求补充，不要直接归因。

优先要素：
- 发生时间（精确到日期/时段）
- 线路/站点/变压器标识
- 电压等级与波动区间
- 持续时长与发生频率
- 是否伴随告警、负载突变、倒闸操作、天气异常

## 何时不阻塞回答
以下场景可直接回答：
- 纯概念解释（如“什么是电压暂降”）
- 用户明确要“通用排查清单”
- 假设题/教学示例

## 结束条件
当用户已补齐关键字段后：
- 立即进入正常回答流程；
- 不重复要求同一批信息；
- 缺少非关键字段时可带假设继续。
